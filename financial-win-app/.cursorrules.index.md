# Índice de Cursor Rules

Este proyecto utiliza archivos `.cursorrules` especializados para diferentes áreas del desarrollo. Cada archivo contiene reglas y convenciones específicas para su dominio.

## Archivos Disponibles

### 📋 `.cursorrules.backend`
**Para**: Desarrollo del backend (NestJS)  
**Contiene**:
- Arquitectura y estructura de módulos NestJS
- Convenciones de controladores, servicios y entidades
- Patrones de autenticación y multi-tenancy
- Migraciones TypeORM
- Testing backend con Jest
- Configuración y observabilidad

**Usar cuando trabajes en**:
- `apps/api/src/`
- Controladores, servicios, módulos
- Entidades TypeORM
- Migraciones de base de datos

---

### 🎨 `.cursorrules.frontend`
**Para**: Desarrollo del frontend (React)  
**Contiene**:
- Arquitectura y estructura de componentes React
- Patrones de React Query (TanStack Query)
- Routing con React Router
- Styling con Tailwind CSS
- Testing frontend con Vitest
- Estado local y global (Zustand)

**Usar cuando trabajes en**:
- `apps/web/src/`
- Componentes React
- Hooks personalizados
- Páginas y rutas
- Styling y UI

---

### 🗄️ `.cursorrules.database`
**Para**: Base de datos y TypeORM  
**Contiene**:
- Estructura de entidades TypeORM
- Convenciones de migraciones
- Patrones de consultas y repositorios
- Multi-tenancy y soft deletes
- Índices y optimización
- Transacciones

**Usar cuando trabajes en**:
- Entidades TypeORM
- Migraciones
- Consultas a base de datos
- Esquemas de base de datos

---

### 🧪 `.cursorrules.testing`
**Para**: Testing (backend y frontend)

- Unit tests con Jest (backend)
- E2E tests con Supertest
- Component tests con Vitest (frontend)
- Test utilities y helpers
- Coverage y thresholds
- Mejores prácticas de testing

**Usar cuando trabajes en**:
- Tests unitarios (`.spec.ts`, `.test.tsx`)
- Tests E2E (`.e2e-spec.ts`)
- Configuración de testing
- Test utilities

---

## Cómo Usar

### Opción 1: Cursor lee automáticamente todos los archivos
Cursor puede detectar y leer múltiples archivos `.cursorrules*` automáticamente. Si estás trabajando en un área específica, Cursor puede referirse al archivo correspondiente.

### Opción 2: Referencia manual en el chat
Cuando trabajes en un área específica, puedes mencionar en el chat:
- "Según las reglas de backend..."
- "Siguiendo las convenciones de frontend..."
- "De acuerdo a las reglas de testing..."

### Opción 3: Archivo principal (recomendado)
Si prefieres un solo archivo principal, puedes crear un `.cursorrules` en la raíz que importe o referencia estos archivos, o Cursor puede leerlos todos automáticamente.

## Recomendaciones

1. **Lee el archivo relevante** antes de empezar a trabajar en un área
2. **Consulta el archivo de convenciones** cuando tengas dudas sobre formato o estilo
3. **Revisa el archivo de seguridad** antes de implementar autenticación o validación
4. **Sigue las reglas de testing** al escribir tests
5. **Consulta el checklist** en cada archivo antes de hacer commit

---

## Actualización de Reglas

Estos archivos deben mantenerse actualizados con:
- Cambios en el stack tecnológico
- Nuevas convenciones del equipo
- Mejores prácticas aprendidas
- Actualizaciones de frameworks

Si encuentras inconsistencias o quieres proponer mejoras, actualiza el archivo correspondiente.

---

## Soporte

Si tienes dudas sobre qué reglas aplicar:
1. Consulta el archivo específico de la área en la que trabajas
2. Revisa el archivo `.cursorrules.code-conventions` para dudas generales
3. Pregunta al equipo o revisa la documentación del proyecto en `docs/`
